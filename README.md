## One Enough Enchantment

本模组允许你动态控制附魔刷新权重甚至完全从游戏中移除附魔。

### 权重控制

原版附魔权重机制详见：
https://zh.minecraft.wiki/w/%E9%99%84%E9%AD%94%EF%BC%88%E7%89%A9%E5%93%81%E4%BF%AE%E9%A5%B0%EF%BC%89
https://minecraft.wiki/w/Enchanting_mechanics

一言概之，权重决定附魔出现频率，而在原版中权重只能由固定的四种稀有度COMMON(10)、UNCOMMON(5)、RARE(2)、VERY_RARE(1)控制。

1.21以后附魔改由数据驱动，权重可以改动，但依旧不够便捷。

本模组提供了游戏中动态控制附魔权重的方法，并允许将权重设置为0进而在游戏中完全移除某个附魔。

在配置文件的rules字段中填入格式化规则，即可控制附魔刷新权重，例如填["minecraft:knockback=100", "minecraft:sharpness=0"]会这样：
- 击退附魔的刷新概率将极大增加。
- 锋利附魔将在游戏中被完全移除。
- 其余附魔不受影响，权重默认为原始权重。

本模组还提供了与KubeJS联动的事件用于动态修改权重，可与游戏阶段等设定配合使用。
```javascript
OEEEvent.modifyWeight(event => {
    event.getWeight("minecraft:sharpness");
    event.setWeight("minecraft:knockback", 0);
});
```

### 移除附魔

默认情况下，由本模组将权重改为0的附魔，不会以任何方式出现在游戏中。

以下配置项默认开启，如果你希望保留部分功能，可以选择性关闭。

配置项dropRecipe：
- 对于权重为0的附魔，移除关联的配方如铁砧配方，这意味着对应的附魔书将无法使用，但仍可以通过指令等方式为物品附魔。
- JEI中将查询不到对应的配方如铁砧配方。

配置项deepDelete：
- 对于权重为0的附魔，即使通过指令等手段强制为物品附魔，对应的附魔数据也无法被解析，但NBT保留着，重新调整权重后可以恢复。
- JEI或创造模式物品栏中对应的附魔书会被移除，完全查询不到有关条目。

### 其他

- 理论上，本模组可以只在服务端安装。
- 使用/oee all指令可以获取所有附魔的信息，包含附魔id、本地化名称、权重。
- 如果你删除了大量COMMON级别的附魔，附魔等级低的附魔台可能会刷新不出附魔，原版机制如此，并非本模组的问题。
